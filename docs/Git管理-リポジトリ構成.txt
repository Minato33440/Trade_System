Gitコマンド分解ガイド
まず && について
複数のコマンドを一行でつなぐ記号。「前のコマンドが成功したら次を実行」という意味。
cd フォルダ && git コマンド（例：cd "C:/Python/REX_AI/Trade_System" && git remote -v ）

Ⅰ. まず大きな流れ
add: 袋に入れる→ commit: 封をして記録 → push: 倉庫へ送る→ [GitHub: 共有の倉庫]→ pull: 倉庫から受け取る
push と pull は逆方向。push＝「自分 → 倉庫」、pull＝「倉庫 → 自分」

①add 「これを次のコミットに含めます」と宣言する
ファイルを編集しても、addしないとGitは無視する。段ボール箱に荷物を入れる作業。
git add note.md　  ← 指定ファイルだけ
git add .　　　　　　← 変更した全ファイル
よく使うのは git add .（ドット）。全部まとめて箱に入れる。

②commit 「この状態を記録します」と確定する
addした変更を1つの「セーブポイント」として保存。
この時点ではまだローカルPC上の記録。GitHubにはまだ届いていない。
git commit -m "バックテスト：エントリー条件を修正"   ← "   " でメモを書いておく
-m "..." はメモ書き。何を変えたか分かる言葉を書く習慣をつけると後で助かる。

③push 「ローカルの記録をGitHubに送る」
commitはあくまでPC内の記録。pushしてはじめてGitHubに届いて、スマホや他の環境から見えるようになる。
git push origin main　← これでちゃんとGitに記憶される
pushしないと「自分のPCにしか存在しない状態」。スマホから見えない。

④pull 「GitHubの最新をローカルに取り込む」
スマホアプリやリモートでの変更をPCに反映させる。作業開始前の「おはようございます」。
git pull origin main　← PCやリモートデバイスにGitに記録ファイルが反映
fetchとmergeを同時にやってくれるショートカット。まずこれだけ覚えておけばOK。

〇裏技
①am「add と commit をまとめる」
git commit -am "修正内容のメモ"　← -a フラグ追加(-am)で add と commit を一発で済ませられる。
新規ファイルには使えないが、既存ファイルの編集なら毎回これでOK。
最後にgit push origin main ← これをやって初めてGitHubに届く
②2ステップに短縮（既存ファイル編集時）
git commit -am "メモ" && git push origin main
※注意点：新規ファイルには使えない。既存ファイルの編集のみ。
-a オプション：既にtrackedされているmodifiedファイルのみを自動ステージング
untrackedファイル（新規作成ファイル）は無視されてこう表示されるdd" to track)
〇 正しい手順
untrackedファイルを含む場合は、以下のいずれかの方法：
方法1: 明示的にaddしてからcommit
git add logs/gm/daily/2026/gm_report/2026-3-16.txt 
git commit -m "メモ"
git push origin main
方法2: 全ての変更をaddしてからcommit　➡　これが一番確実👍
git add .
git commit -m "メモ"&& git push origin main
方法3: 個別にaddしてから -a でcommit
git add logs/gm/daily/2026/gm_report/2026-3-16.txt
git add logs/gm/daily/2026/gm_report/2026-3-17.txt
git commit -am "daily report up to date"&& git push origin main
※事前確認：今後同様の状況を避けるため、コミット前に 
git status でステージング状態を確認することをおすすめします。
③amp「エイリアス機能」で一括コミット＆プッシュフラグ作成
git config --global alias.amp '!git commit -am "$1" && git push'
これ一回設定しておくと：
git amp "エントリー条件を修正"　← 一発でcommit＆pushできる。
Minatoのシステムみたいに個人運用でサクサク回したい用途にはぴったりだと思う。
ClaudeCodeに設定させちゃえば30秒で終わるよ。

● GitHubからpushしたデータを削除する
git rm -r docs/Sistem_logs/
リポジトリ（インデックス）と作業ディレクトリの両方からディレクトリとファイルを即座に削除
• 削除後、git commit で変更を確定（コミットしないとリモートには反映されない）
• 作業ディレクトリのファイルは完全に消える
git rm --cached
次のコミットでリポジトリ(インデックス)からファイルを削除（リポジトリから削除の準備）
• 作業ディレクトリのファイルは残る
• 次回の git commit でリポジトリから削除される
• ディレクトリの場合: git rm --cached -r <ディレクトリ> でインデックスからディレクトリを削除
git restore --staged <ファイルf or フォルダー> 
ステージングエリア（インデックス）からファイルの変更を取り消すコマンド（予約取り消し）
注意：これは「リポジトリからファイルを削除する」コマンドではありません。
何が起きるか
•  git add <ファイル> でステージした内容を取り消す（アンステージ）
•  作業ディレクトリの実際のファイルはそのまま残る
•  次の git commit にその変更は含まれなくなる
間違いやすい点
•  git rm --cached <ファイル> に近い動きで、--staged は「ステージ済みの変更を戻す」
• ファイル自体を削除するなら git rm（もしくは rm + git add）を使います
つまりmainリポジトリに上げた（コミット済みの）ステージ状態のキャンセルです。

●ブランチの削除方法
git branch -d [ブランチ名]：ローカルブランチを削除するコマンドです。
	• 例：git branch -d feature/test
	• マージ済みのブランチのみ削除できます（未マージの場合はエラー）。
	• 強制削除したい場合は -D を使います（git branch -D [ブランチ名]）。
実行例：
cd "c:\Python\REX_AI\Trade_System" && git branch -a
cd "c:\Python\REX_AI\Trade_System" && git checkout main && git branch -d claude/rex-ai-system-overview-uYU2X && git push origin --delete claude/rex-ai-system-overview-uYU2X && git push origin --delete claude/document-team-structure-JdGmZ
cd "c:\Python\REX_AI\Trade_System" && git branch -D claude/rex-ai-system-overview-uYU2X && git push origin --delete claude/rex-ai-system-overview-uYU2X && git push origin --delete claude/document-team-structure-JdGmZ
cd "c:\Python\REX_AI\Trade_System" && git branch -a


● Gitアカウント移設方法
状況：
	• ローカルPC + 旧GitHubアカウントに UCAR_DIALY が存在
	• 新GitHubアカウントへ UCAR_DIALY だけ移設したい
手順はこの3ステップ
ステップ1：旧アカウントのリポジトリをバックアップ用にクローン
cd C:\Python
git clone --mirror https://github.com/旧アカウント名/UCAR_DIALY.git UCAR_DIALY_backup.git
--mirrorで履歴含めて完全バックアップ。
ステップ2：新アカウントにリポジトリを作る
GitHubの新アカウントにログインして、UCAR_DIALYという名前で空のリポジトリを作成。
 READMEやgitignoreは追加しないこと（空っぽで作る）。
ステップ3：新アカウントのremote URLを確認
新GitHubアカウントのUCAR_DIALYリポジトリのURLをコピーしておく。
ステップ4：ローカルのremote URLを新アカウントに向け直す
VS Codeのターミナルで：
cd C:\Python\UCAR_Dialy
git remote set-url origin https://github.com/新アカウント名/UCAR_DIALY.git
ステップ5：確認
新GitHubアカウントのページでコミット履歴が全部来てればOK。
ステップ6：force pushで上書き
git push origin main --force
新アカウントのREADMEとgitignoreは上書きされるけど、バックアップがあるので問題なし。
ステップ７：完全ミラーにしたい場合は、この方法が最もシンプルです(バックアップ用にクローンを再現）
git clone --mirror https://github.com/setona2006/UCAR_DIALY.git ucar_mirror
cd ucar_mirror
git remote set-url origin https://github.com/Minato33440/UCAR_DIALY.git
git push --mirror
