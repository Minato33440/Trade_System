  Step3: エントリー中は毎バー check_5m_swing_confirmed() を実行
         → True になった瞬間に swing_confirmed = True へ更新

# 4間押し目検出→5分足エントリーまでのロジック


4時間ダウ切り上げ(戻り高値越え)から4時間Swing-HighLow検出後、

1時間ごとに、過去4時間分の15分足プロップチャートを作成
目的：正確な値位置判断を視覚情報として検証するため＋ロジック設計の明確化

具体案：プロップでチャートから以下の形状と値位置を検出
①15分ダブルボトム15分ネックライン越えを検出
②15分ダブルボトムサポートが4時間上昇波に対してFibo61.8以内に存在
目的：4時間が押し安値を割っていない条件で→15分足上昇3波を狙う
　　　＝これまでと違うロジックだが構造がより明確化されるため検知ミスリスクが軽減

何故1時間ごとのプロップ作成か？
15分ダブルボトムネックライン越えから再度押し目を作り始めるまで最低4～6本は必要なため。

エントリーロジックへ移行
・15分ダブルボトムネックライン越えから15分足の直近Swing-HighLowを検出
・5分足ダブルボトムネックラインが15分足Swing-Highの値位置以下である条件で
　→5分足ダブルボトムネックライン越え実体確定の次の足でエントリー(＝同ロジック)

決済ロジック：
・初動上昇波の決済は15分ダウ崩れに集約
・5分足Swing-HighLow検出後は、これまでと同じ
1時間ネックライン到達まで5分ダウ崩れ決済
4時間ネックライン到達で半値決済
4時間ネックライン1時間足実体確定後は15分ダウ崩れ決済

Q1. 「1H ネックライン」の定義
エントリー前の直近1H Swing High か、エントリー後に形成されるものか？
→ 実装上の取得タイミングが完全に異なる。
・回答： 「1H ネックライン」の定義は、エントリー前の直近1H Swing Highでいいよ。
ただ、4時間ネックラインを越える前も越えた後も、エントリ―決済基準が「15分Swing-HithLow」と「5分足Swing-HighLow」しかないため、最初は１HSwing-HighLowロジックをなくしてシンプル化してもよいかと。
　→つまり、1Hワンボトムでもダブルボトムでも15分ロジックが効けばエントリーするという形。
　→なぜなら、１Hワンボトムは大体15分ダブルボトム位の時間軸になることが多いため）
(将来的に、勝率と損益比率を見て４時間ダブルネック到達前に１Hダブルネック到達で部分利確ロジックを入れ込むでもいいかもね。）

Q2. 「4H ネック + 1H 実体確定」の解釈
「1H足のCloseが4H Swing Highを越えた足」の理解で正しいか？
それとも「4H足自体のCloseが4H SHを越えた確定」か？
回答：基本的に１H実体上抜け確定でいいよ。
抽出するのはシンプルに4時間Swing-Highを、何時00分から5分足12本目の足が上抜け実体確定（15分足4本目）と同じ。
(例：5時丁度に１H足が４Hネック上抜け確定なら＝5時丁度に15分足＆5分足上抜け確定と同じ)

Q3. Prop chartの用途
backtest時のデバッグ用PNG保存か、リアルタイム用の監視ツールか？
→ 実装の優先度と複雑度が変わる。
回答：Prop chartは主にbacktest用途で、Swing-HighLowの抽出からエントリー決済の値位置等を視覚化することでロジック整合性や細かな値位置調整ができるため、先ずはこの「プロット機能」の作成を優先したいところ。
基本的に5分足プロットから各時間足すべてののHighLow構造を形状的に抽出できるかと。

Q4. 「15M ダブルボトム」と「15M ネックライン」の使い分け
DBの2つの安値の間にある反発高値 = DBネックライン。
これが「15Mネックライン越え」の判定ラインと同一か？
それともSwing High検出で別途取得するか？
回答：「DBの2つの安値の間にある反発高値 = DBネックライン」を５分足実体上抜け確定でよいかと。
（５分足実体上抜けだと早期エントリーになり損切が多く成る様なら→ロジック調整でネック＋Xpipsで実体確定にしてもいいかも）
