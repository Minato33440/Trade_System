## 2. トリガー条件（プロット生成のタイミング）

以下の全条件が揃った時点で 1 枚の PNG を生成・保存する。

```
① 4H 上昇ダウ確認
     Swing High 切り上がり AND Swing Low 切り上がり → 'LONG'

② 現在価格が Fib 61.8% 圏内（≤65% バッファ含む）
     優位性★★★: Fib 45〜55%（Fib50 + 4H ネックライン付近）
     優位性★★ : Fib 65% 以内

③ 15M 前回押し目安値を割っていない
     上位構造（4H 上昇ダウ）が維持されている確認

④ 15M ダブルボトム形成確認
     SL① → 反発 → SL② ≧ SL① の構造
  15M ダブルボトム + 5M DB 成立確認
    15M: SL① → 反発 → SL② ≧ SL①（15M DB）
    5M : 2番底安値が 15M 前回 Swing Low の -X pips 以内
         （X = 0 でスタート、config の WICKTOL_PIPS で調整）
    WICKTOL_PIPS = 0.0   # 初期値（許容なし）
    # → 5, 10 の順でテスト

⑤ 5M DB ネックライン上抜け実体確定
     5M の Open/Close の低い方（実体下端）がネック上に確定した足
     ← この瞬間にプロット生成・保存
```

⑤ 前（④だけ成立）の段階では保存しない。
理由：⑤なしは 15M 下落トレンド継続の誤検知を拾うリスクがあるため。

## 5. 表示期間

```
メインパネルの x 軸範囲:
  エントリー時刻を中心に
  左側: 4 時間分（5M 足 48 本）
  右側: 2 時間分（5M 足 24 本）
  合計: 約 6 時間分（72 本）を標準表示

調整オプション:
  バックテスト完了後に全件一括出力する場合は
  エントリー起点から決済まで全範囲を自動拡張
```
