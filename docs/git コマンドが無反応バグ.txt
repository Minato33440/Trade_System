## バグ報告：PowerShell で git コマンドが無反応になる問題

### 環境
- OS: Windows 11 Pro
- Shell: PowerShell (VSCode ターミナル)
- 作業ディレクトリ: `C:\Python\REX_AI\Trade_System`

---

### 症状

```powershell
PS C:\Python\REX_AI\Trade_System> git remote -v
PS C:\Python\REX_AI\Trade_System> git log --oneline -3
PS C:\Python\REX_AI\Trade_System>   # 出力なし・エラーなし
```

---

### 原因

PowerShell の PATH に**偽の `git` ファイルが2つ**存在しており、本物の Git より先に呼ばれていた。

| ファイル | サイズ | 問題 |
|---|---|---|
| `C:\Windows\System32\git` | 0バイト | **主犯**。System32 は PATH 優先度が高いため常にこれが呼ばれた |
| `C:\Python\REX_AI\git` | 0バイト | 副犯。リポジトリ整理作業中に誤作成されたと推定 |
| `C:\Program Files\Git\cmd\git.exe` | 正常 | 本物。上記2つに隠されて呼ばれていなかった |

> **なぜエラーではなく無反応だったか：** 0バイトの実行ファイルは何もせず終了するため、エラーメッセージも出力も発生しなかった。

---

### 修正手順

```powershell
# 1. REX_AI 直下の偽 git 削除（通常権限）
Remove-Item C:\Python\REX_AI\git

# 2. System32 の偽 git 削除（管理者権限で実行）
Remove-Item C:\Windows\System32\git

# 3. 本物の git が認識されているか確認
where.exe git
# → C:\Program Files\Git\cmd\git.exe のみ表示されればOK
```

---

### 修正後

```powershell
PS C:\Python\REX_AI\Trade_System> git remote -v
origin  https://github.com/Minato33440/Trade_System.git (fetch)
origin  https://github.com/Minato33440/Trade_System.git (push)

PS C:\Python\REX_AI\Trade_System> git log --oneline -3
c895624 (HEAD -> main, origin/main) add logs/gm data and update .gitignore
2c311dc first commit
```

**ローカル `.git/config` 自体は最初から正常**だったため、クローンし直しは不要だった。
