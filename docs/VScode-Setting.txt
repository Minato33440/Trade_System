●  新規ターミナル起動時に自動で入る（Cursor側のSetting）
1. CursorSettiinng→General→Editor Setting設定で Activate Environment または python.terminal.activateEnvironment を検索
2. 表示されるチェックを ON／OFF（ユーザー と ワークスペース 両方で ）
3. Cursor を開き直すか、新規ターミナルを作り直して echo $env:VIRTUAL_ENV を確認
補足：PowerShell に "python.terminal.activateEnvironment": false をコマンドとして打つのはNGで、これは 設定ファイル/設定UIで指定する値です（前にエラーになってたやつ）。

● 新規ターミナル起動時に自動で入る（推奨：PowerShell Profile）
4. まず PowerShell でプロファイルを開く：
notepad $PROFILE
5. 中に、これを追加（プロジェクト直下をカレントにしてくれてる前提）：
$activate = Join-Path -Path (Get-Location) -ChildPath ".venv\Scripts\Activate.ps1"
if (Test-Path $activate) { . $activate }
6. もし Cursor が起動時のカレントをプロジェクト直下にしない場合は、絶対パス版が確実です：
. "C:\Python\REX_AI\Trade_System\.venv\Scripts\Activate.ps1"
※ セキュリティ設定で止まる場合は「PowerShell 実行ポリシー」調整が必要になることがあります
